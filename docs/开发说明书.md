# 文心识典·论语专版
## 前端打包
$env:VITE_API_BASE_URL="/api"; npm run build

## 后端启动
python3安到了: cd /usr/local/src/Python-3.12.4
后端服务配置文件:
sudo vi /etc/systemd/system/wenxin-backend.service.内容如下
[Unit]
Description=Wenxin Python Backend Service
After=network.target

[Service]
User=root
WorkingDirectory=/opt/wenxin/backend-prod/app
Environment="PATH=/opt/wenxin/backend-prod/app/venv/bin"
ExecStart=/opt/wenxin/backend-prod/app/venv/bin/uvicorn main:app --host 127.0.0.1 --port 8000

[Install]
WantedBy=multi-user.target

启动并设置开机自启：
sudo systemctl daemon-reload
sudo systemctl start wenxin-backend
sudo systemctl enable wenxin-backend
sudo systemctl status wenxin-backend
### Nginx 配置
sudo vi /etc/nginx/conf.d/wenxinclass.conf
server {
    listen 9000;                # 改为 9000 端口，避开备案检测
    server_name 47.104.224.19;  # 直接使用你的服务器公网 IP

    # 前端静态文件
    location / {
        root /opt/wenxin/upload/dist;
        index index.html index.htm;
        try_files $uri $uri/ /index.html; 
    }

    # 后端 API 代理
    location /api/ {
        proxy_pass http://127.0.0.1:8000; # 对应你跑在 3.12 环境下的后端
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
}
2. 测试并重载 Nginx：
sudo nginx -t
sudo nginx -s reload

在阿里云后台放行 9000 端口（最关键）
即使服务器内部配置好了，如果阿里云的外部防火墙（安全组）没开，你还是访问不到。
登录 阿里云控制台。
找到你的 ECS 实例。
在左侧菜单点击 “安全组”。
点击当前使用的安全组 “配置规则”。
在 “入方向” 点击 “手动添加”：
协议类型：TCP
端口范围：9000/9000
授权对象：0.0.0.0/0 (表示允许所有人访问)
点击 “保存”。

检查服务器本地防火墙 (如果有)
有些服务器系统自带了 firewalld，也需要放行一下：
### 检查防火墙状态
sudo systemctl status firewalld

如果显示 active (running)，则需要运行以下命令放行端口：
sudo firewall-cmd --zone=public --add-port=9000/tcp --permanent
sudo firewall-cmd --reload

浏览器直接访问

http://47.104.224.19:9000
