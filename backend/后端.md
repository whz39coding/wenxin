# 文心识典·论语专版 后端开发详细说明书
## 一、后端基础信息
### 1. 项目全称
文心识典·论语专版 - 后端服务中心
### 2. 核心定位
基于**FastAPI**的轻量化B/S后端服务，为前端7大功能提供**RESTful API接口**，承载**用户管理、文件处理、OCR解析、RAG知识库、智能修复**核心逻辑，所有数据本地化运行，无云端依赖。
### 3. 固定技术栈（无歧义、大学生零门槛）
| 分类                | 技术工具                          |
|---------------------|-----------------------------------|
| 后端框架            | FastAPI + Uvicorn（异步服务）|
| 业务数据库          | MySQL 8.0（存储用户/上传/记录）|
| 向量知识库          | Chroma（本地轻量向量库，论语专属） |
| ORM映射             | SQLAlchemy（数据库操作）|
| 身份认证            | PyJWT（登录Token校验）|
| AI核心引擎          | LangChain（RAG框架）|
| 文字识别            | PaddleOCR（论语古籍OCR专用）|
| 大模型/向量模型     | Qwen-1.8B-Chat + bge-small-zh(本地部署下载)     |
| 配置管理            | python-dotenv（环境变量）|
| 自动接口文档        | FastAPI Swagger UI（内置）|

### 4. 开发核心原则
1. **分步开发**：从基础框架→业务功能→AI核心
2. **模块化解耦**：每个模块独立开发、独立测试
3. **数据专一**：开发期**仅使用《论语》前5篇**测试，不冗余
4. **轻量运行**：CPU可跑，通过API key调用LLM,无需GPU.

## 二、后端代码项目结构（标准模块化架构）
采用**Python包管理规范**，文件夹=独立模块，结构清晰、团队协作友好，所有文件/包作用明确：
```
backend/                     # 后端根目录
├── main.py                  # 项目唯一入口文件（启动服务）
├── .env                     # 全局环境变量（数据库/模型路径）
├── config/                  # 【模块1】全局配置包
├── core/                    # 【模块2】核心框架包
├── api/                     # 【模块3】API路由包（所有前端接口）
├── models/                  # 【模块4】数据库模型包
├── services/                # 【模块5】核心业务服务包（最核心）
│   ├── auth_service.py      # 用户认证服务
│   ├── file_service.py      # 文件上传服务
│   ├── ocr_service.py       # OCR解析服务
│   ├── rag_service.py       # RAG知识库/检索服务
│   ├── repair_service.py    # 智能修复服务
│   └── word_service.py      # 生僻字解析服务
├── utils/                   # 【模块6】通用工具包
├── data/                    # 本地数据包（论语文本/向量库）
└── tests/                   # 接口测试包
```

## 三、后端核心模块详细说明（6大核心包 + 子模块）
每个模块明确：**核心功能、使用技术、包含文件/子包**
运行命令：`uvicorn main:app --reload`

### 模块1：全局配置包（config/）
**核心功能**：统一管理项目所有配置，避免硬编码，全局复用
**使用技术**：python-dotenv（环境变量读取）
**包含文件**：
1. `settings.py`：核心配置文件（数据库地址、Chroma路径、JWT密钥、OCR模型参数、词嵌入大模型路径,LLM API KEY等配置信息）
2. `constant.py`：常量定义（《论语》篇目列表、允许上传的文件格式、生僻字字典）
等等
### 模块2：核心框架包（core/）
**核心功能**：后端底层基础能力，支撑整个项目运行
**使用技术**：FastAPI 核心类、JWT 加密、异常处理
**包含文件**：
1. `exceptions.py`：全局异常处理（文件上传失败、OCR报错、RAG无结果）
2. `security.py`：JWT认证核心（Token生成、校验、权限验证）
3. `database.py`：MySQL数据库连接引擎、Chroma向量库初始化
等等
### 模块3：API路由包（api/）
**核心功能**：对接前端，定义所有接口路由，**前端点击菜单→调用对应接口**
**使用技术**：FastAPI APIRouter（路由分组）
**包含文件/子模块**：
1. `auth.py`：用户登录/注册接口
2. `file.py`：文件上传/OCR解析接口
3. `rag.py`：语义检索/知识库接口
4. `repair.py`：古籍智能修复接口
5. `word.py`：生僻字解析接口
6. `hall.py`：3D展厅数据接口
7. `user.py`：个人中心接口
等等
### 模块4：数据库模型包（models/）
**核心功能**：定义MySQL数据表结构，ORM映射
**使用技术**：SQLAlchemy ORM
**包含文件**：
1. `user.py`：用户表（账号、密码、信息）
2. `upload.py`：上传记录表（用户上传的论语图片/OCR结果）
3. `repair.py`：修复记录表（残缺文本→修复结果）
4. `dictionary.py`：《论语》生僻字字典表
等等
### 模块5：核心业务服务包（services/）→ 项目灵魂
**核心功能**：所有业务逻辑/AI逻辑的实现层，API只调用服务，不写逻辑
**使用技术**：LangChain、PaddleOCR、Qwen大模型、Chroma
**子模块拆分（独立开发）**：
1. `auth_service.py`：用户注册、登录、密码加密
2. `file_service.py`：文件保存、格式校验、本地存储
3. `ocr_service.py`：PaddleOCR加载、论语图片文本提取
4. `rag_service.py`：论语文本向量化、知识库构建、语义检索
5. `repair_service.py`：残缺文本匹配、AI补全、修复依据生成
6. `word_service.py`：生僻字查询、通假字解析、例句匹配
等等
### 模块6：通用工具包（utils/）
**核心功能**：全项目复用的工具函数，无业务逻辑
**使用技术**：Python原生、Pillow（图片处理）
**包含文件**：
1. `file_util.py`：文件格式校验、图片压缩
2. `text_util.py`：论语文本清洗、断句、格式标准化
3. `model_util.py`：AI模型加载工具（避免重复加载）
等等

## 四、后端分阶段开发规划
严格按照**基础→业务→AI**顺序开发，每阶段完成后，用《论语》前5篇测试验证。

### 阶段1：基础架构搭建
**开发目标**：搭建后端骨架，启动服务，连接数据库
**开发模块**：config/ + core/ + main.py
**核心工作**：
1. 配置环境变量、数据库连接参数
2. 初始化FastAPI服务、MySQL连接、Chroma向量库
3. 编写项目入口文件，启动服务，访问自动接口文档
**交付成果**：可运行的后端服务，数据库连接成功
**测试标准**：浏览器打开 `http://localhost:8000/docs` 看到Swagger接口文档

### 阶段2：用户认证模块开发
**开发目标**：实现前端登录/注册功能
**开发模块**：models/user.py + api/auth.py + services/auth_service.py
**核心工作**：
1. 创建用户数据表
2. 开发注册、登录接口
3. 实现JWT Token生成与校验
**交付成果**：登录/注册接口可调用
**测试标准**：注册用户、登录获取Token，接口校验通过

### 阶段3：文件上传模块开发
**开发目标**：实现前端《论语》PDF上传功能
**开发模块**：models/upload.py + api/file.py + services/file_service.py + utils/file_util.py
**核心工作**：
1. 创建上传记录表
2. 开发文件上传接口、格式校验、本地存储
**交付成果**：前端上传图片→后端保存成功
**测试标准**：上传《论语·学而篇》图片，后端成功存储

### 阶段4：OCR解析模块开发
**开发目标**：实现论语图片→文本提取
**开发模块**：services/ocr_service.py + api/file.py + utils/text_util.py
**核心工作**：
1. 集成PaddleOCR古文模型
2. 开发OCR解析接口，文本清洗格式化
**交付成果**：上传论语图片→返回纯文本
**测试标准**：上传学而篇图片，OCR输出「学而时习之，不亦说乎」

### 阶段5：RAG知识库构建模块开发 → 核心基础
**开发目标**：将《论语》前5篇存入向量库，构建专属知识库
**开发模块**：services/rag_service.py + core/database.py + data/
**核心工作**：
1. 加载bge-small-zh向量模型
2. 导入《论语》前5篇纯文本，向量化存入Chroma
3. 初始化本地向量库
**交付成果**：论语专属向量库构建完成
**测试标准**：向量库可检索到学而篇、为政篇文本

### 阶段6：智能语义检索模块开发 → 核心亮点
**开发目标**：实现前端自然语言检索《论语》
**开发模块**：api/rag.py + services/rag_service.py
**核心工作**：
1. 集成Qwen-1.8B大模型
2. 开发RAG语义检索接口
3. 结果格式化（原文+译文+出处）
**交付成果**：前端提问→返回论语对应内容
**测试标准**：提问「什么是仁」，返回论语中对应原文

### 阶段7：智能修复 + 生僻字模块开发
**开发目标**：实现残缺文本修复、生僻字解析
**开发模块**：
1. 智能修复：api/repair.py + services/repair_service.py
2. 生僻字：models/dictionary.py + api/word.py + services/word_service.py
**核心工作**：
1. 残缺文本匹配RAG知识库，AI补全文字
2. 录入《论语》专用生僻字字典，开发查询接口
**交付成果**：残缺文本修复、生僻字解析接口可用
**测试标准**：输入「学而时习之，不亦__乎」→ 自动补全「说」

### 阶段8：3D展厅数据 + 全功能联调
**开发目标**：支撑前端3D展厅，所有接口联调
**开发模块**：api/hall.py + 所有模块联调
**核心工作**：
1. 开发3D展厅篇目数据接口（论语20篇列表）
2. 所有接口统一校验、异常处理优化
**交付成果**：后端全功能接口可用
**测试标准**：前端7大菜单功能全部调用后端接口成功

## 八、开发提示
1. 后端先定义接口

- 例如 POST /api/favorites

2. 前端在 API 层新增调用函数

- 新建 frontend/src/api/favorites.ts
- 在 index.ts 导出

3. 页面层调用

- 在对应页面如 Profile.tsx 导入 favorites API 函数并调用
- 页面不直接拼接 axios URL，统一走 src/api

4. 本地联调

- 前端通过 Vite 代理把 /api 转发到后端（你已配置）
- 前端拿到 access_token 后会自动带 Authorization 头（你已在 client.ts 做了拦截器）

你接下来可以直接做的两步

1. 启动后端后，用前端登录页 Auth.tsx 测试注册和登录闭环。  
2. 选一个小功能（例如收藏）我带你按同样分层再完整走一遍，这样你会真正熟悉“后端放哪写、前端怎么调、如何联调排错”的完整节奏。
